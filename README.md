# 🚦 Smart City Traffic Forecasting using Machine Learning

## 📌 Project Overview

The Smart City Traffic Forecasting System is a Data Science and Machine Learning project developed to predict traffic volume at different city junctions based on historical traffic patterns.

The project uses Exploratory Data Analysis (EDA), Feature Engineering, and a Random Forest Regressor model to forecast future traffic conditions. An interactive Streamlit web application has been developed to allow users to predict traffic volume by entering date, time, and junction information.

---

## 🎯 Objectives

* Analyze historical traffic data.
* Identify traffic patterns and trends.
* Predict traffic volume using Machine Learning.
* Provide an easy-to-use web interface for traffic forecasting.
* Support smart city traffic management and planning.

---

## 📊 Dataset Information

The dataset contains historical traffic information collected from multiple city junctions.

### Features:

| Feature  | Description                  |
| -------- | ---------------------------- |
| DateTime | Date and Time of observation |
| Junction | Junction Number              |
| Vehicles | Number of vehicles observed  |
| ID       | Unique Record Identifier     |

### Total Records

* 48,120 rows
* 4 original features

## 🔍 Exploratory Data Analysis

The following analyses were performed:

### 1. Hourly Traffic Analysis

* Traffic is lowest during early morning hours.
* Traffic increases significantly during daytime and evening hours.

### 2. Junction Analysis

* Junction 1 experiences the highest traffic volume.
* Junction 4 experiences the lowest traffic volume.

### 3. Monthly Trend Analysis

* Traffic volume varies across months.
* Peak traffic was observed during certain periods of the year.

## 🛠 Feature Engineering

The DateTime column was converted into useful features:

* Day

## 📈 Model Performance

| Metric   | Value |
| -------- | ----- |
| MAE      | 2.40  |
| MSE      | 12.63 |
| R² Score | 0.969 |

The model achieved an R² Score of 96.9%, indicating excellent prediction performance.

Note:
The trained model file (traffic_model.pkl) is not included in this repository because it exceeds GitHub's web upload size limit. It can be regenerated by running the training script (src/train_model.py) using the provided dataset.

## 🌐 Streamlit Application

The project includes an interactive Streamlit application where users can:

* Select Junction
* Enter Year
* Select Month
* Select Day
* Select Hour

The application predicts expected traffic volume and categorizes traffic as:

* Low Traffic
* Moderate Traffic
* Heavy Traffic
